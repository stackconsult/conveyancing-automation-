## Project Structure

- memory/legal_knowledge/ - Property laws, regulations, precedents
- memory/document_templates/ - Legal contracts, forms, standard documents
- memory/process_workflows/ - Conveyancing procedures, compliance checklists
- memory/stakeholder_data/ - Client preferences, agent coordination patterns
- memory/risk_factors/ - Legal/financial risks, mitigation strategies
- memory/compliance_rules/ - Regulatory requirements, validation rules
- memory/market_data/ - Property values, trends, analytics
- memory/api_reference/ - Mem0 API documentation and examples
- memory/implementation_patterns/ - Code patterns and implementation guides
- memory/architecture_patterns/ - System design and architecture patterns
- memory/best_practices/ - Development guidelines and best practices
- memory/testing_strategies/ - Testing frameworks and validation procedures
- memory/error_handling/ - Troubleshooting guides and error resolution

## Integration Points

### Agent-Memory Mapping

- Document Analysis Agent → legal_knowledge, document_templates
- Compliance Agent → compliance_rules, risk_factors

- All memories must include metadata for traceability
- Legal memories must include jurisdiction information
- Document templates must include version control
- Process workflows must include validation checkpoints
- Risk factors must include probability and impact assessments
